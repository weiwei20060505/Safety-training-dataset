【陷阱 1：維度災難 (Curse of Dimensionality)】

問題狀況：
  • 樣本數 N = 50
  • 特徵維度 d = 1024
  • 比例 N/d = 0.049 (極端危險！)
  • 結果：100% 訓練準確率，但 50% 測試準確率

修正方案：
  ✓ 增加樣本：50 → 2000
  ✓ 改進後 N/d = 1.95 (安全範圍)
  ✓ 加入 L2 正則化：C=0.01 (LogisticRegression, SGDClassifier)
  ✓ 特徵標準化：StandardScaler

預期改善：
  • 訓練準確率：95% → 70-80% (下降但更實誠)
  • 測試準確率：50% → 65-75% (大幅提升！)
  • 泛化能力：大幅改善

───────────────────────────────────────────────────────────────────────────────

【陷阱 2：類別不平衡 (Severe Class Imbalance)】

問題狀況：
  • 假設基礎模型準確率 90%
  • 在 100 筆中，錯誤樣本只有 10 筆
  • 模型傾向永遠預測「沒有錯誤」
  • 結果：90% 準確率，但 Recall = 0%（無法檢測錯誤）

修正方案：
  ✓ 啟用 class_weight='balanced'
  ✓ 自動增加稀有類（錯誤）的權重
  ✓ 應用到所有 5 種模型（SGD, LR, MLP, RF, LGB）

預期改善：
  • Accuracy：90% → 75-80% (下降是因為停止瞎猜)
  • Recall：0% → 60-75% (真的能檢測到錯誤了！)
  • Precision：N/A → 85% (保持高質量)
  • ROC AUC：0.5 → 0.7-0.8

───────────────────────────────────────────────────────────────────────────────

【陷阱 3：數據洩漏 (Data Leakage)】

問題狀況：
  • 用所有 100 筆資料訓練基礎模型
  • 再用「相同的 100 筆」產生 y_err 標籤
  • 基礎模型「背答案」了
  • 結果：訓練時 95%，上線後 60%

修正方案：
  ✓ 使用 5-fold 交叉驗證
  ✓ 確保每筆預測時基礎模型都沒看過該樣本
  ✓ 產生客觀的 y_err 標籤

預期改善：
  • 訓練/測試差異：<5% → 20-30% (差異大但合理)
  • 上線準確率：60% → 70% (真實表現)
  • 結果可信度：↑↑↑
